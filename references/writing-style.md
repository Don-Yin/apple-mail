# Email Writing Style Guide

Guidelines for composing professional emails via the Apple Mail skill.

## General Principles

- **Concise and direct** — get to the point quickly
- **Professional tone** — avoid slang, maintain courtesy
- **Clear structure** — use short paragraphs, one idea per paragraph
- **Action-oriented** — state what you need and by when

## Formatting

- Use plain text (Mail.app drafts use plain text content)
- Separate paragraphs with blank lines
- Use dashes or asterisks for lists, not bullet Unicode characters
- Keep line length reasonable (no hard wraps needed, Mail.app handles this)

## Reply Style

- Quote the relevant part of the original message, not the entire thread
- Place your response above the quoted text
- Use "------- Original Message -------" separator (automatic in reply-draft)

## Forward Style

- Add context at the top explaining why you're forwarding
- Use "------- Forwarded Message -------" separator (automatic in forward-draft)

## Common Patterns

### Acknowledgment
```
Thanks for sending this over. I'll review and get back to you by [date].
```

### Request
```
Could you please [action] by [date]? Let me know if you need any additional context.
```

### Follow-up
```
Following up on [topic] from [date]. Have you had a chance to [action]?
```

## Things to Avoid

- All caps (reads as shouting)
- Excessive exclamation marks
- Reply-all when only the sender needs to see the response
- Forwarding without adding context
- Sending without reviewing the draft first (always use compose-draft then send-draft)
- Never use double dashes (`--`) in email text
